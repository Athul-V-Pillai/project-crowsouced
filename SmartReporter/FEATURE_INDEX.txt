 FEATURE HIGHLIGHTS
================================================================================

 Forgot Password Functionality
   - Users can request password reset
   - Email sent with secure reset link
   - Works for both admin and client users

 Secure Token System
   - Tokens generated using crypto.randomBytes(32)
   - Hashed with SHA-256 before storage
   - Expire after 1 hour
   - Single-use only

 Password Reset Page
   - Real-time password strength indicator
   - Confirmation password validation
   - Success/error messages
   - Auto-redirect to login

 Email Integration
   - Uses nodemailer for email sending
   - Gmail support with app passwords
   - Test mode for development (console logging)
   - Custom HTML email template

 Security Features
   - Token hashing (SHA-256)
   - Password hashing (bcrypt)
   - Email verification
   - 1-hour token expiration
   - Automatic token cleanup
   - Backend token validation
   - Frontend token verification

 SECURITY CHECKLIST
================================================================================

 Token Security
   Generated from crypto.randomBytes(32)
   Hashed with SHA-256 before storage
   Expires after 1 hour
   Single-use only
   Cleared after successful reset
   Validated on every request

 Password Security
   Validated for minimum length (6 chars)
   Hashed with bcrypt (10 salt rounds)
   New password different from old
   Never logged or displayed
   Securely transmitted over HTTPS (prod)

 Email Security
   Link includes secure token
   Token embedded in URL
   Email not logged
   Uses TLS/SSL for transmission
   Gmail app passwords (not account password)

 API Security
   Input validation on all endpoints
   Email validation (format check)
   Token validation (format & expiry)
   User existence verification
   No user enumeration attacks
   Error messages don't reveal email status

 Database Security
   Tokens never stored in plain text
   Passwords always hashed
   Timestamps for audit trails
   Old tokens automatically cleared
   No sensitive data in logs

 TROUBLESHOOTING REFERENCE
================================================================================

PROBLEM: Email not being sent
SOLUTIONS:
  1. Check EMAIL_USER is set in .env
  2. Check EMAIL_PASSWORD is set in .env
  3. For Gmail: Ensure App Password is used, not account password
  4. For Gmail: Enable 2-Step Verification first
  5. Restart backend server after .env changes
  6. Check backend console for error messages

PROBLEM: Reset link not working
SOLUTIONS:
  1. Check token hasn't expired (1 hour limit)
  2. Verify URL has complete token
  3. Check frontend is running on port 3002
  4. Check backend is running on port 5000
  5. Clear browser cache and try again
  6. Request new reset link

PROBLEM: Token verification fails
SOLUTIONS:
  1. Clear browser cache
  2. Check MongoDB connection
  3. Verify token is not expired
  4. Check backend server is running
  5. Check API response in browser console (F12)
  6. Check backend console for errors

PROBLEM: Can't login after reset
SOLUTIONS:
  1. Check password reset was successful
  2. Verify new password was saved to database
  3. Check for extra spaces in password
  4. Check caps lock is off
  5. Try password reset again
  6. Check MongoDB for user record
