FEATURE OVERVIEW
================================================================================

The password reset feature allows both admin and regular users to securely reset 
their forgotten passwords through an email-based verification system.

Key Capabilities:
  ✅ Forgot password functionality
  ✅ Secure token-based reset links
  ✅ 1-hour token expiration
  ✅ Password strength indicator (frontend)
  ✅ Support for both admin and client users
  ✅ SHA-256 token hashing
  ✅ bcrypt password hashing
  ✅ Email notifications (with fallback console logging)

HOW TO USE
================================================================================

SETUP (One-time):
  1. Restart backend server: npm run dev
  2. (Optional) Configure email in backend/.env
  3. Restart frontend server: npm run dev

USER FLOW:
  1. User visits: http://localhost:3002
  2. Clicks "Forgot Password?" link
  3. Enters email address and submits
  4. Checks email for reset link (or console log if not configured)
  5. Clicks link or manually navigates to reset page
  6. Enters new password with strength indicator
  7. System validates and resets password
  8. User redirected to login
  9. User logs in with new password

================================================================================
CONFIGURATION OPTIONS
================================================================================

OPTION 1: TEST MODE (No Email Setup)
  - Leave EMAIL_USER and EMAIL_PASSWORD empty
  - Emails will be logged to backend console
  - Useful for development and testing

OPTION 2: GMAIL (Recommended)
  - Generate App Password: https://myaccount.google.com/apppasswords
  - Set EMAIL_USER=your-email@gmail.com
  - Set EMAIL_PASSWORD=your-16-char-app-password
  - Requires 2-Step Verification enabled

OPTION 3: CUSTOM EMAIL SERVICE
  - Edit backend/src/utils/mail.js
  - Replace nodemailer configuration with your provider
  - Examples: SendGrid, Mailgun, AWS SES, etc.

================================================================================
API ENDPOINTS
================================================================================

1. REQUEST PASSWORD RESET
   POST /api/auth/forgot-password
   Request:  { "email": "user@example.com" }
   Response: { "success": true, "message": "Email sent..." }

2. VERIFY RESET TOKEN
   POST /api/auth/verify-reset-token
   Request:  { "resetToken": "token-value" }
   Response: { "success": true, "message": "Token is valid" }

3. RESET PASSWORD
   POST /api/auth/reset-password
   Request:  { 
     "resetToken": "token-value",
     "newPassword": "new-password-123"
   }
   Response: { "success": true, "message": "Password reset successfully" }

================================================================================
SECURITY FEATURES
================================================================================

 Tokens are hashed using SHA-256 before database storage
 Tokens expire after exactly 1 hour
 Tokens are invalidated after successful reset
 Passwords are hashed using bcrypt with salt
 Reset link is single-use only
 Email verification prevents unauthorized resets
 Backend validates token integrity on every request
 Frontend validates email format before submission
 CORS protection on all endpoints

================================================================================
TROUBLESHOOTING
================================================================================

Problem: Email not being sent
Solution: 
  - Check EMAIL_USER is configured in .env
  - Check EMAIL_PASSWORD is set (or use test mode)
  - Restart backend server after .env changes
  - Check backend console for error messages

Problem: Reset link not working
Solution:
  - Verify token has not expired (1 hour limit)
  - Check URL has complete token value
  - Clear browser cache
  - Request new reset link

Problem: Token validation fails
Solution:
  - Ensure backend is running on port 5000
  - Check MongoDB connection is active
  - Verify reset token is not expired

Problem: Password validation fails
Solution:
  - Ensure password is at least 6 characters
  - Verify both password fields match
  - Check for special characters/spaces
